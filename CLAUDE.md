# Whim

**タイトルは「Whim」**（2026-07-11 改名。旧称: よりみちクエスト）。"on a whim"＝ふとした思いつきで。英語表記のまま使い、カタカナにしない。候補 Whim / Detour からユーザーが Whim を選択。ディレクトリ名・GitHubリポジトリ名も 2026-07-12 に whim へ変更（公開URLは t11086.github.io/whim に移動。localStorage は同一オリジンなので記録は引き継がれる）。**localStorage キー（yorimichi_v1）だけは旧名のまま**（変えると既存ユーザーの記録が消えるため今後も変えない）。

## 企画概要

**ターゲット**: 大人（20〜40代）の気分転換。毎日が仕事と家事のループになっていて、決めた予定をこなすだけの一日になっている人。子どもと一緒に遊んでも成立する。

**コンセプト**（2026-07-12 転換。旧: 「役に立たない小さなクエスト」）:
「明日、傘がいるかどうかを調べる」「八角を買ってくる」「左手の感情線を見る」「ダンボールを丸く切る」——
**今日やる予定じゃなかったことが、毎日1枚のカードで出てくる**アプリ。
役に立つかどうかは問わない。**問うのは「予定になかったかどうか」だけ**。
気の利くリマインダーも、知らなかったことの発見も、小さな工作も、
「予定になかった」という一点で同じカードに乗る。

**キャッチコピー**: 「予定になかったことを。」（旧: 「人生に、意味のない用事を。」）
※数を宣言しない（「ひとつ」と言わない）。リロールも1日の消化数も無制限なので、数を言うと仕様と食い違う

### コンセプト転換の経緯（2026-07-12）

旧コンセプトは「アンチ生産性 ＝ 役に立たないことをする」だったが、**実際に作ったものが先にそこから離れていた**。拡充バッチ1で却下された7本（アルミホイル球・紙を小さく折る・硬貨の穴のぞき・遠いものに命名 等）は、すべて「純粋に無意味なもの」。逆に採用・実装されたのはスパイス系チェーン14本、せかいのキッチン、手相・舌診、点検系6本——どれも無意味ではない。コンテンツが投票済みだったので、遅れていたコンセプトのほうを合わせた。

**この転換で撤回された前提**:
- 「アンチ生産性」という自己規定
- 「為になるものは2〜3割まで」という比率制限（→ 為になる系は3本柱の1本なので、上限を設ける理由がない）

**転換しても変わらないもの（重要）**:
- **文体ガイドは丸ごと生きる**。とくに「効能を書かない」は絶対に維持する。効能を書かない理由が「無意味だから」から「**そのほうが手が動くから**」に変わっただけ。ここを緩めると「天気を確認して明日に備えよう」的な文体が入り込み、世に無数にある生活改善アプリと同じ棚に並ぶ
- ストリークなし・責めない・リロール無制限・自己申告・30秒〜5分・お金がかからない

**設計原則**:
- 効能を謳わない。「リラックスできます」「集中力が上がります」等の文言をUIに一切出さない
- ストリーク（連続日数）なし。何日サボっても何も起きない、責めない
- 完了は自己申告でOK。証明を求めない
- クエストは30秒〜5分で終わり、お金がかからないものだけ（スパイス等の買い物ステップは例外的に少額の出費あり）

## コアループ

1. アプリを開くと**クエストカードが1枚だけ**表示される（3枠案はユーザー判断で1枚に変更、2026-07-09）
2. 気に入らなければ**リロール（引き直し）無制限**。ボタンでもカードの左右スワイプでも引ける
3. やったら「やった！」で**即完了**。ダイアログなし、絵文字バーストの軽いエフェクトだけで次のカードが出てくる（＝1日の消化数は無制限。写真添付は2026-07-10に廃止）
4. **テキスト入力があるクエストは、カード上に入力欄が出る**（「やった」を押す前に入力。入力は任意、アルバムに残る。完了時の入力ダイアログは2026-07-10に廃止）

### リロール無制限の採用について（決定事項）

無制限を採用する。理由: リロールに制限を付けると「もったいないから引けない」という圧が生まれ、ゆるさというコンセプトの核と矛盾する。また「石を拾う」は都心では無理、「ダンボールを切る」は手元にないと無理、といった実行可能性のばらつきを吸収する安全弁がリロールなので、ここをケチると詰みが発生する。

ただし「プール全体を眺めてタスクを消費し尽くす」暴発を防ぐため、以下でガードする:

- **1枚ずつ、スライドアニメーション付きで引く**（一覧性を与えない。カードを引くガチャ感が自制を促す）
- **その日リロールで流したクエストは、同日中は再登場しない**
- **デッキは日替わりシャッフル**（日付シードで決定的に生成。「見尽くした感」の起点を作らない）
- クエスト一覧画面は作らない。プールの中身は遊びながらしか見えない

### チェーンクエスト（つづきもの）（2026-07-10 追加）

複数のクエストを順にこなすと何かが完成する仕掛け。**完成して初めて意図に気づく**のが狙いなので、正体は徹底的に隠す:

- チェーンの**現在のステップだけ**が通常クエストに混ざってデッキに入る（一度に1枚）
- 見た目は普通のカード。カテゴリチップなし、チェーンであることを示すUIなし
- ステップを完了すると、次のステップがデッキの**ランダムな位置に紛れ込む**（直後には出ない）
- 買い物系ステップ（八角・シナモン等）には**「もう家にある」ボタン**が出る。押すと完了扱い
- 完成の演出は `finale` 付きステップのダイアログのみ（「なにかが完成した！」スタンプは2026-07-10に廃止）。ダイアログは**カードが飛ぶ前に表示**され、とじると完了してカードが飛ぶ（短歌のお披露目も同じ順序）
- **チェーンのカードは連続して出ない**（直前がチェーンなら次はチェーン以外を優先して引く）
- **同じチェーンの次のステップは、間に3枚はさまるまで出ない**（クールダウン。完了・リロールで1枚消化するごとに1減る。デッキが尽きたときだけ例外的にゆるむ）（2026-07-10 追加）
- リロールで流された場合は通常クエスト同様、同日中は再登場せず翌日また混ざる

### せかいのキッチン（国旗コレクション）（2026-07-12 追加、ユーザー発案）

料理系チェーンは「他国の家庭にある定番を少しずつ体験するシリーズ」。チェーンに `country: { flag, name, dish, note }` を付けると:

- 最終ステップの完了時に**国旗つきダイアログ**が自動で出る（スタンプ「◯◯、完成。」＋国旗シール＋「◯◯の家庭の定番です。〜」）。ステップ側に `finale` を書く必要はない（CHAIN_STEPS 生成時に自動付与）
- アルバム上部の**「せかいのキッチン」**に国旗スタンプ（国旗＋料理名＋国名）がたまる。保存はせず S.chains の進行状況から導出（「戻す」で最終ステップを取り消すと消える、で整合）
- 国旗は手貼りシール風の小さなインラインSVG（FLAGS に9か国: 台湾・中国・エジプト・モロッコ・インド・メキシコ・フランス・アメリカ・トルコ）。**絵文字の国旗は使わない**（絵文字全廃方針）
- コースター・短歌・マニキュア星は国なし（料理ではないので対象外）

### かいものメモ（2026-07-12 追加、ユーザー発案）

買い物ステップは「いますぐは行けない」ことがある。そのための逃し方。**「あとでやるリスト」ではなく「かいものメモ」として作る**のが設計の要（未完了タスクが積み上がるリストは罪悪感の発生装置で、ストリークなし・責めないという背骨と正面から矛盾する。**モノのリストなら圧が出ない**。「八角」とメモに書いてあるのは買い物中に見つけて嬉しいもの、「八角を買ってくる（未完了）」は借金）。3本柱の「あとで効くこと」に乗る機能。

- 対象は **`have: true` の買い物ステップだけ**（29本）。他のカードには出さない。汎用の「キープ」は作らない
- カードの「かいものメモに入れる」で**そのカードはデッキから消える**（もう出てこない）。チェーンは進まない＝買うまでそのチェーンは止まる
- メモの □ をタップ＝買った → そのステップが完了し、チェーンが進んでアルバムにも残る（カードを引かずに完了する唯一の経路）
- × でメモから外す → カードとしてデッキに戻るだけ。買わなくても何も起きない
- **圧をかけないガード**: 件数バッジを出さない・赤くしない・「残りN件」と言わない。**上限6件**（頭に入る量を超えたらメモではなく雑用リスト。上限に達すると「メモに入れる」ボタンが出ない）
- 常設タブは作らない。**1件以上あるときだけ**ヘッダー右上に入口が出て、タップでダイアログ（0件ならUIごと存在しない。せかいのキッチンと同じく状態から導出）
- 「クエスト一覧画面は作らない」の原則には抵触しない（あの原則はプールの中身を隠すためのもので、メモに載るのは自分が引いて選んだカードだけ）
- ボタンが増えて混むので、「かいものメモに入れる」は**テキストボタン**（波線下線・くすみブルー）。ベタ塗りは「やった」だけに保つ
- よそで同じ品が手に入るとメモからも自動で消える（`pruneMemo`）

### 買った品はチェーン間で共有する（2026-07-12 バグ修正）

**シナモンは4チェーン（台湾風醤油・五香粉・ほうじ茶チャイ・ゴールデンミルク）、八角・クミン・パプリカパウダー・オレガノ・カルダモンは2チェーンずつで、別々に「買ってくる」ステップを持っている。** 修正前は台湾風醤油のためにシナモンを買っても五香粉があとで再び「シナモンを買ってくる」と要求していた（「もう家にある」で救えるが、ユーザーが覚えている前提だった）。

- **持ち物 ＝ スパイス申告ぶん ＋ これまでに買ったぶん**（`hasItem`）。買った記録は **`S.log` から導出する**（`boughtItems`。保存しないので「戻す」でログを消せば自動で持ち物からも消え、整合が崩れない）
- 買い物ステップを完了すると（「もう家にある」も同じ）、全スパイス系チェーンに `ownedSkip` をかけ直し、位置がずれた古いカードをデッキから整理する（`syncChainCards`）
- 「戻す」で買い物を取り消すと、その品を読み飛ばしていた**他のチェーンもまとめて巻き戻る**（ログから位置を数え直し → 残りの持ち物ぶんだけ読み飛ばし直す）
- 既存ユーザーの過去の買い物も、load() 時にログから導出されるのでさかのぼって効く

### スパイス申告（2026-07-10 追加）

- クエスト40「家にあるスパイスを、ぜんぶ教えて」: カード上に17種のスパイスチップ（八角・シナモン・花椒・クローブ・フェンネル・クミン・コリアンダーパウダー・ごま・カルダモン・パプリカパウダー・オレガノ・タイム・ローズマリー・ターメリック・ブラックペッパー・ガーリックパウダー・オニオンパウダー。後半4種は2026-07-12追加）が並び、持っているものをタップで選んで「きめた」。選択内容はアルバムに残る。**一度答えたらもう出ない**（S.spices に保存。null = 未申告）
- **申告が終わるまで、スパイス系チェーン10本（台湾醤油・デュカ・五香粉・ほうじ茶チャイ・タコス・プロヴァンス・ゴールデンミルク・BBQポップコーン・シナモンコーヒー・カルダモンコーヒー）はデッキに入らない**（買い物カードの洪水対策を兼ねる）。ミントティー・コースター・短歌・マニキュア星は封印対象外
- **申告後にアップデートで増えたスパイス系チェーン**: load() 時に申告済みなら全スパイス系チェーンへ ownedSkip をかけ直すので、持っているスパイスの買い物ステップは自動で読み飛ばされる。位置がずれた古いステップカードは load() の stale 掃除でデッキから除去（場に出ていたら引き直し）。申告リストに無い新スパイス（ターメリック等）は「持っていない」扱いになるが、「もう家にある」ボタンで救済できる
- 申告後は、持っているスパイスの「買ってくる」ステップを**自動でスキップ**してチェーンが始まる（チェーン途中の買い物も、進行時に持っていれば読み飛ばす）。ステップの `item` フィールドとスパイス名を照合。「もう家にある」ボタンは救済として残る
- クエスト文は理由を説明しない謎カードのままにする（チェーンの正体を予告しないため）
- アルバムの「戻す」で申告を取り消すと未申告に戻る: チェーン6本を再封印してデッキから回収、自動スキップぶんの進行を取り消し（実際に完了したステップはログから数え直して維持）、申告カードをデッキに再挿入（2026-07-11 修正）
- 封印中（未申告）のチェーンステップを「戻す」した場合、進行位置は巻き戻すがカードはデッキに戻さない（申告→ステップの順で全部戻しても、封印中のカードがデッキに紛れない）（2026-07-11 修正）

※「ダンボールを丸く切る」「好きなN文字を考える」は通常プールから外して各チェーンの1歩目に移動（重複防止）
※ペース調整（1日1ステップまで等）は今後の検討課題

## 継続の仕掛け（責めないコレクション）

- **おもいでアルバム**: 完了したクエストが日付ごとに並ぶ。テキスト入力（本のタイトル、好きな文字）や完成した短歌もここに残る（写真添付は2026-07-10に廃止）。各項目に「戻す」ボタンあり（デバッグ用。完了を取り消し、同日ならdoneTodayからも除去、チェーンは最新ステップなら1歩巻き戻す）。リスト最下部に「はじめから」ボタン（デバッグ用の全リセット。確認ダイアログ→localStorage削除→リロード）（2026-07-11 追加）
- **むいみ週報**: 週に1回、無意味な統計を表示。「今週あなたは石を2つ拾いました」「猫の絵: 通算3枚」。真面目なグラフは出さない
- **称号**: 同カテゴリの累積で解锁（「すべすべ鑑定士」「ダンボール加工技師」など）。通知や達成圧はかけず、アルバムの隅にひっそり増える
- 進捗はローカル保存（localStorage）

## タスク文体ガイド（★ここがプロダクトの本体）

タスクの温度感がすべて。量産時に自己啓発臭・雑用臭に退化させないためのルール:

1. **数を指定する** — 「ひとつ」「1枚」「10歩」。数が入ると急にクエストらしくなる
2. **質感・条件をひとつ足す** — 「すべすべの」「まっすぐな」「利き手じゃないほうの」。この一語が詩になる
3. **効能を書かない** — 「〜してリラックス」は禁止。やる理由を説明しない
4. **言い切りの動詞で終える** — 「〜しよう」「〜してみませんか」ではなく「拾う」「切る」「描く」
5. **3本柱のどれかに入るものだけを出す**（次項）

- 良い例: 「明日、傘がいるかどうかを調べる」「冷蔵庫をあけて、あと1回ぶんしかないものを探す」
- 悪い例: 「深呼吸してリラックスしよう」（効能臭）／「部屋を掃除する」（大きい・義務・雑用）／「アルミホイルで、できるだけまんまるの球を作る」（文体は満点だが、やっても何も起きない）

※旧ガイド5は「微妙に為になるものを2〜3割混ぜる」だった。コンセプト転換（2026-07-12）で「為になる系」は3本柱の1本になったので、**比率の上限は撤廃**

### タスクの3本柱（★2026-07-12 コンセプト転換で確定。量産時はこれでふるいにかける）

出すのは「今日やる予定じゃなかったこと」。**役に立つかどうかは問わない**。ただし、やったあとに**何かが残る**必要がある。残り方が3種類ある:

**1. あとで効くこと**（気の利くリマインダー）— *あとの自分の行動が変わる*
- 例: 明日、傘がいるかどうかを調べる／冷蔵庫をあけて、あと1回ぶんしかないものを探す／財布の中の現金を数える／スマホの充電が何％のこっているか見る／明日履く靴を、玄関に出しておく／玄関の靴をぜんぶそろえる／レシートを出す
- ユーザーの言葉では「**やってよかった、と後から思える小さな行動**」。詩的な意図はなくてよい

**2. まだ知らないこと**（発見）— *知らなかったことを知る*
- 例: スパイスの使い方（せかいのキッチン、チェーン14本）／手相占い・舌診（知らなかった自分）／氷を口でとかす（知らなかった体感）
- スパイスは「知らない味」、手相は「知らない自分」、氷は「知らない感覚」。**知る対象は外にも内にもある**

**3. 手元に残ること**（つくる）— *モノが残る*
- 例: コースター／短歌／猫の絵／マニキュア星／五香粉などの完成物
- チェーンクエストの主戦場

**却下の基準**: 3本柱のどれにも入らないもの。すなわち **やっても何も残らないもの**——観念的な遊び（〜に名前をつける）、無目的な工作（紙を小さく折る）、何も変わらないのぞき見（硬貨の穴をのぞく）、アルミホイル球。「あまりにも意味がないものは無くしたい」（ユーザー、2026-07-12）

**「見るだけ」の線引き**（重要）: 「見るだけだから却下」ではない。分かれ目は**見た結果、あとの自分の行動が変わるか／知らなかったことを知るか**。硬貨の穴をのぞいても何も起きない（却下）が、冷蔵庫をのぞくと買い物が変わる（柱1）、手相をのぞくと知らない自分を知る（柱2）。

**柱1を書くときの文体上の落とし穴**: このタイプはうっかり効能を書きたくなる（「〜を確認して備える」「買い忘れを防ぐ」）。**効能は絶対に書かず、見る行為だけを言い切る**（ガイド3・4）。「明日の天気を確認する」ではなく「明日、傘がいるかどうかを調べる」。「**確認する**」という動詞自体が事務的で効能が透けるので使わない。また、数を数えさせる形（何ロール・何％・いくら）にするとクエストらしくなる（ガイド1）。

|---|---|

| どうぐ | なし / 紙とペン / はさみ類 / スマホ |

**UIにはタグもフィルタも一切出さない**（2026-07-09、「極力シンプルに」の方針でおうちモードと「ばしょ」表示は実装後に廃止）。タグはデータとして保持し、将来のバランス調整用。

一部のクエストは**カード上のテキスト入力欄**に書き込める（例: 32「本を1冊選ぶ」→本のタイトル、短歌チェーンの各ステップ）。「やった」を押す前に入力する方式（2026-07-10、完了ダイアログから変更）。入力は任意で、アルバムにクエスト文と一緒に残る。

**二択クエスト**（2026-07-10 追加）: 「やった」の代わりに選択肢ボタンが出て、選んだ結果に応じたメッセージが表示される仕組み。例: 21「左手の感情線は〜のびていますか？」→「のびてる／のびてない」→ 手相占いの結果「あなたは〜な人です。」が出て、アルバムにも残る。**先に結果ダイアログを表示し、とじたらカードが飛んでいく**順序（2026-07-10 修正）。手相占いは21〜25の5パターン（感情線・運命線・頭脳線・親指の組み方・小指の長さ）。**占いカードは連続して出ない**（直前が占いなら次は占い以外を優先して引く。他に札がないときだけ例外）。**セッションの1枚目にも出ない**（アプリを開いた直後の表示が占いになる場合は別のカードに差し替え。占いはデッキに残る）。

**カメラクエスト**（2026-07-10 追加）: 26「今日履いている靴の写真を撮る」は「やった」の代わりに「📷 しゃしんを撮る」ボタンが出て、その場でカメラが起動。撮影すると縮小（長辺640px, JPEG）してアルバムに写真つきで残り、そのまま完了になる（汎用の写真添付は廃止のまま。カメラはこのタイプのクエスト限定）。アルバムでは72pxのサムネイル表示で、タップすると拡大表示（ライトボックス）になる。

## デザイン方針（2026-07-11 刷新）

「韓国の감성手帳アプリ」「10〜20代のSNS」「手書き感」がテーマ。旧デザイン（クリーム地＋テラコッタ＋丸角＋絵文字、いわゆるClaudeで作った感）は全廃。

- **紙**: 白ベース（#f4f3ee）にドット方眼。カードとダイアログは白い紙＋マスキングテープ（黄。山札は青/ピンク）＋薄い1px罫線、角丸は小さめ（4〜8px）
- **文字**: 日本語は Zen Kurenaido（Google Fonts、ペン字風）で全環境統一。オフライン時はシステム手書き系（Klee → 教科書体 → 丸ゴ）に落ちる。**ロゴ「Whim」は Bradley Hand Bold をアウトライン化したインラインSVG**（2026-07-11、ユーザーが Bradley Hand の字形を好んだため画像化して全環境統一。フォント読み込み不要）。少し傾けて、下にキャッチコピーを小さく添える。つうさんの数字は Caveat（英語手書き）
- **UI文言のルール**（2026-07-11）: **「クエスト」というゲーム用語をUIに出さない**（ユーザー意向）。呼ぶときは「ようじ」（タブ「きょうのようじ」、アルバム「つうさん N ようじ」）。リロールボタンは「スキップ」（旧「べつのにする」）。コード内の識別子・コメント・このドキュメントでは従来どおりクエストと呼んでよい
- **色**: モノクロ（インク #33312c／サブ #9b978e）＋差し色はくすみブルー #5c74a8 のみ。完了ボタンは黒ベタ
- **絵文字はUIに一切使わない**: タブは線画SVG、完了エフェクトは ✦✧＋ の手描き記号、アルバムの印は ✓（CSSで付与）。CATS の emoji/color はデータとして残っているが未使用
- **手書きディテール**: 「べつのにする」は波線下線のテキストボタン、入力欄は下線だけ（手帳の罫線）、アルバムの写真はポラロイド風（白フチ＋傾き）、スタンプはくすみブルーの傾いた枠
- **読ませたいサブ文字は `--sub2`（#6b675e）を使う**（2026-07-12）。`--sub`（#9b978e）は薄すぎて読みづらいとユーザー指摘。「スキップ」「かいものメモ」入口・コーチマークは `--sub2`。装飾的な文字（日付ラベル、タグライン等）は従来どおり `--sub`
- **常設のヒント文を置かない**（2026-07-12）: 「← よこに スワイプしても スキップ →」は冗長なので廃止。**初回起動時にカードを左右に揺らし（`.wiggle`）、下に「よこにスワイプしても スキップできます」をコーチマークとして出す**。一度見たら `S.coached` で二度と出ない。カードにさわった時点でも消える（`endCoach`）
- **ダイアログ内のボタンは `.mbox button { width: 100% }` を継承する**ので、小さいボタン（メモの □ / ×）を置くときは **id セレクタで詳細度を上げて打ち消す**（`#memolist button { width: auto }`）。クラスだけだと負けて幅100%のベタ塗りになる

## 技術方針

- 単一HTML + CSS + Vanilla JS（既存プロジェクトと同様、ブラウザで開くだけで動く構成）
- **PWA対応**（2026-07-12）: `manifest.webmanifest`＋`sw.js`＋アイコン（`icon.svg` / `icon-192.png` / `icon-512.png` / `icon-180.png`=apple-touch-icon）でホーム画面追加・standalone起動・オフライン動作に対応。単一HTML方針の例外ファイル群。アイコンはロゴの「W」を白い紙＋黄マステに載せたデザインで、`icon.svg` から headless Chrome で PNG 化して生成。SWはナビゲーションはネット優先（更新反映）、Google Fontsと同一オリジンはキャッシュ優先。`file://` で開いたときはSW登録をスキップするので従来どおりローカルでも動く。キャッシュを更新したいときは `sw.js` の `CACHE` のバージョン番号を上げる
- クエストプールはJS内の静的配列（id, 本文, カテゴリ, ばしょ, どうぐ）
- 日替わりデッキは日付文字列をシードにした決定的シャッフル
- 写真はローカル保存（縮小してlocalStorage、容量が厳しければIndexedDB）
- 外部依存なし、オフライン動作、スマホ縦画面ファースト。**例外はフォントのみ**: Google Fonts（Zen Kurenaido / Caveat）を読み込む。オフライン時・読み込み失敗時はシステムの手書き系（クレー→教科書体→丸ゴ）にフォールバックし、機能は損なわれない（2026-07-11、環境間でフォントを統一したいというユーザー意向）
